Visão Geral do Sistema

O Tem na Cozinha MA é uma aplicação web voltada para a valorização e preservação da culinária maranhense. O sistema permite que qualquer usuário informe quais ingredientes tem disponíveis em casa e receba, em tempo real, uma lista de receitas tradicionais que pode preparar com esses ingredientes — incluindo receitas que estão "quase prontas" (com até 3 ingredientes faltando).

Funcionalidades do Sistema

  • Busca por ingredientes: usuário seleciona ingredientes que tem em casa e recebe receitas compatíveis, divididas em "tenho tudo" e "quase lá"
  • Filtros avançados: categoria, dificuldade, tempo de preparo, ocasiões, opções veganas/diet
  • Seções especiais: receitas organizadas por tema — festa junina, dias de festa, receitas rápidas, cardápio diário
  • Modo guiado: apresentação passo-a-passo do modo de preparo com cronômetro e narração por voz (Web Speech API)
  • Timer integrado: temporizador persistente durante o preparo da receita
  • Sugestão de receitas: formulário para o usuário sugerir novas receitas, enviado por e-mail ao administrador
  • Receitas em destaque: curadoria manual de receitas exibidas na página inicial

O frontend é uma Single Page Application (SPA) construída com React 18 e bundleada com Vite. Utiliza roteamento client-side com React Router e estilização via Tailwind CSS (utility-first). Toda a comunicação com o backend ocorre via Fetch API, encapsulada no módulo src/services/api.js.

Custom Hooks

useReceitas

Faz uma requisição GET /receitas ao montar o componente e normaliza os dados: converte campos booleanos (TINYINT), parseia arrays JSON (ocasioes, modo_preparo) e separa ingredientes obrigatórios dos opcionais.

useFiltros

Hook central da página de busca. Gerencia o estado de todos os filtros ativos (busca por texto, categoria, dificuldade, tempo, ocasiões, flags booleanas e ingredientes). Usa useMemo para recalcular eficientemente todasFiltradas, receitasCompletas e receitasQuase somente quando o estado muda.

useSpeech

Encapsula a Web Speech API nativa do navegador para narração dos passos do modo guiado. Faz a síntese de voz em português (pt-BR) sem dependências externas ou custo.

TimerContext

Contexto global que mantém um cronômetro regressivo ativo mesmo quando o usuário navega entre páginas. Usa useContext + useState + useEffect com setInterval.

Comunicação com a API

O arquivo src/services/api.js encapsula todas as chamadas HTTP. A URL base é definida pela variável de ambiente VITE_API_URL, configurada no Vercel para apontar para https://temnacozinhaapi.duckdns.org.

// src/services/api.js const BASE_URL = import.meta.env.VITE_API_URL || 'http://localhost:3333' export const api = { get: (path) => fetch(`${BASE_URL}${path}`).then(res => res.json()), post: (path, body) => fetch(`${BASE_URL}${path}`, { method: 'POST', headers: { 'Content-Type': 'application/json' }, body: JSON.stringify(body), }).then(res => res.json()), }

Algoritmo de Busca por Ingredientes (Frontend)

O hook useFiltros implementa a lógica de matching localmente no cliente, após ter carregado todas as receitas uma única vez:

  1. Para cada receita, conta quantos ingredientes obrigatórios o usuário possui
  2. Calcula faltando = total_obrigatórios - possuidos
  3. receitasCompletas: faltando === 0 (tem tudo)
  4. receitasQuase: faltando > 0 e faltando < total (tem pelo menos 1)

Decisão de design: todas as receitas são carregadas uma única vez no hook useReceitas e o matching é feito no cliente com useMemo, evitando múltiplas requisições à API a cada tecla digitada e tornando a busca instantânea após o carregamento inicial.

Backend — API REST

O backend é uma API REST construída com Node.js e Express, rodando na porta 3333 dentro de uma instância AWS EC2. Segue o padrão MVC simplificado: rotas → controllers → banco de dados.

Nginx — Proxy Reverso

O Nginx fica na frente da aplicação Node.js, responsável por terminar o TLS (HTTPS) e encaminhar as requisições para a porta 3333. A porta 3333 não é exposta diretamente para a internet.

# /etc/nginx/sites-available/api server { listen 443 ssl; server_name temnacozinhaapi.duckdns.org; ssl_certificate /etc/letsencrypt/live/temnacozinhaapi.duckdns.org/fullchain.pem; ssl_certificate_key /etc/letsencrypt/live/temnacozinhaapi.duckdns.org/privkey.pem; location / { proxy_pass http://localhost:3333; proxy_http_version 1.1; proxy_set_header Upgrade $http_upgrade; proxy_set_header Connection 'upgrade'; proxy_set_header Host $host; proxy_cache_bypass $http_upgrade; } }

PM2 — Gerenciador de Processos

O PM2 garante que a API Node.js continue rodando após reinicializações do servidor, reinicia automaticamente em caso de crash e facilita o monitoramento.

# Iniciar a API com PM2 pm2 start src/app.js --name api # Configurar para iniciar automaticamente no boot pm2 startup pm2 save # Reiniciar recarregando variáveis de ambiente pm2 restart api --update-env # Monitorar em tempo real pm2 logs api

DuckDNS — Domínio Gratuito

O DuckDNS fornece o subdomínio temnacozinhaapi.duckdns.org que aponta para o IP público da EC2. Como o IP pode mudar ao reiniciar a instância, o DuckDNS permite atualizar o IP manualmente pelo site ou via API.

HTTPS e Segurança

Por que HTTPS é obrigatório?

O frontend está no Vercel com HTTPS. Navegadores modernos bloqueiam por padrão requisições de uma página HTTPS para uma API HTTP — isso é chamado de Mixed Content. Sem HTTPS na API, o site não consegue buscar receitas.

Let's Encrypt + Certbot

O certificado SSL gratuito é emitido pela Let's Encrypt via Certbot, utilizando o método de validação HTTP-01 (não DNS-01, para evitar problemas com registros CAA do DuckDNS):

# Instalar Certbot com plugin Nginx sudo apt install certbot python3-certbot-nginx # Emitir certificado (HTTP-01, sem DNS) sudo certbot --nginx -d temnacozinhaapi.duckdns.org \ --preferred-challenges http

Renovação automática: o Certbot instala automaticamente um cron job que renova o certificado antes de expirar (válido por 90 dias). Não é necessária intervenção manual.

Camadas de Segurança da API

CamadaImplementaçãoProteção
Transport HTTPS / TLS 1.2+ Dados criptografados em trânsito
Cabeçalhos HTTP Helmet.js XSS, clickjacking, MIME sniffing
Origem CORS restritivo Apenas frontend autorizado acessa a API
Rate Limiting express-rate-limit Proteção contra DoS e abuso
Injeção SQL Queries parametrizadas (mysql2) Nunca concatena input do usuário no SQL

Credenciais Variáveis de ambiente (.env) Senhas nunca no código-fonte

Consultas Parametrizadas

Todas as queries SQL usam o operador ? do mysql2, que garante que o input do usuário seja tratado como dado (nunca como código SQL):

// CORRETO — parametrizado, imune a SQL injection pool.query('SELECT * FROM receitas WHERE id = ?', [id]) // ERRADO — concatenação vulnerável (NÃO usado no projeto) pool.query(`SELECT * FROM receitas WHERE id = ${id}`)

Funcionalidades Principais

Busca por Ingredientes

O principal diferencial do sistema. O usuário seleciona os ingredientes que tem em casa usando o componente IngredientePicker, que faz autocomplete consultando GET /ingredientes/buscar?q=. Os resultados são divididos em:

  • Tenho tudo: receitas cujos ingredientes obrigatórios o usuário possui todos
  • Quase lá: receitas onde faltam entre 1 e (total-1) ingredientes obrigatórios

Ingredientes opcionais (marcados com opcional=1 no banco) são exibidos na receita mas não são contados no algoritmo de matching, pois o usuário pode fazer o prato sem eles.

Modal de Detalhes da Receita

Ao clicar em uma receita, o componente RecipeDetailModal exibe todos os detalhes: imagem, ingredientes (destacando os que o usuário tem e os que faltam), modo de preparo em lista numerada e botão para iniciar o modo guiado.

Modo Guiado

O componente ModoGuiado apresenta os passos do modo de preparo um a um, com:

  • Navegação por passos (anterior / próximo)
  • Barra de progresso visual
  • Narração por voz usando a Web Speech API (sem custo, nativa do navegador)
  • Integração com o MiniTimer para temporização de passos

Sugestão de Receitas

O formulário em /sugerir envia um POST /sugestoes com os dados da receita sugerida. O controller valida todos os campos, sanitiza contra XSS usando escapeHtml() e envia um e-mail HTML formatado para o administrador via Gmail SMTP com Nodemailer.

Rate limiting de sugestões: cada IP pode enviar no máximo 5 sugestões por hora, prevenindo spam no e-mail do administrador.

Seções Especiais

A página /secoes agrupa as receitas em categorias temáticas configuradas estaticamente no arquivo src/data/secoes.js (não requer chamada de API adicional):

  • Receitas Rápidas (<30 minutos)
  • Para Festas (tipica_festa = true)
  • São João / Festa Junina
  • Cardápio do Dia a Dia
  • Vegano / Sem Carne

Fluxos de Dados

Fluxo 1: Busca por ingredientes

1
Carregamento inicial useReceitas faz GET /receitas → recebe todas as receitas com ingredientes → armazena no estado React
2
Usuário digita ingrediente IngredientePicker faz GET /ingredientes/buscar?q=... → exibe sugestões de autocomplete
3
Usuário seleciona ingredientes adicionarIngrediente() atualiza o estado de filtros no useFiltros
4
useMemo recalcula instantaneamente receitasCompletas e receitasQuase são recalculados no cliente sem nova requisição à API
5
RecipeGrid exibe os resultados Seções "Tenho tudo" e "Quase lá" com badges indicando ingredientes faltando

Fluxo 2: Modo guiado de preparo

1
Clique na receita Abre RecipeDetailModal com os dados já carregados (sem nova requisição à API)
2
Clique em "Modo guiado" Renderiza ModoGuiado com o array modo_preparo da receita
3
Narração automática por voz useSpeech chama window.speechSynthesis.speak() com o texto do passo atual em pt-BR
4
Timer persistente MiniTimer lê/escreve no TimerContext global, que persiste mesmo ao navegar entre páginas

Fluxo 3: Sugestão de receita

1
Formulário preenchido Estado local gerenciado com useState em SugerirReceita.jsx
2
POST /sugestoes api.post('/sugestoes', form) via Fetch API com Content-Type: application/json
3
Validação + sanitização no servidor Campos obrigatórios, tamanhos, formato CPF/e-mail, escapeHtml() contra XSS
4
Envio de e-mail Nodemailer envia e-mail HTML formatado ao administrador via Gmail SMTP com App Password
5
Feedback visual Frontend exibe tela de sucesso com confirmação ao usuário
